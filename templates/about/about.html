More information about Pnyx

Overview

Pnyx is a user-friendly and yet powerful application that supports the whole process of collective decision making (from setting up a poll/election to eliciting individual preferences to aggregating them in a reasonable way to the communication of outcomes to participants). Pnyx takes into account the insights that have been generated in social choice theory (including new insights regarding probabilistic methods) and does so in a highly user-friendly way without expecting knowledge about the properties of the underlying mechanisms.

The features of a poll

Pnyx polls have several functional features to suit a variety of different use case scenarios while not overwhelming the user with a large amount of customization options.

Naming and description

Name: The name of the poll. It is used mostly for poll management purposes.

Question: The main question to be resolved with the poll. It is visible on the ballot.

Description: A description of the poll. It is visible on the ballot.

Visibility

Public poll: The poll is visible to anyone with knowledge of its unique URL. Any visitor can vote and see the results.

Private poll: The poll is visible only to invited participants. Each participant receives a unique link for his own ballot and can only cast one vote. In addition to invited participants the poll admin can see the results, too.

Change vote: For private polls this attribute allows the participants to change their votes before the closing time of the poll. By default each participant can cast his vote only once.

Temporary results: If enabled, it allows all participants to see temporary results at any time (even before the poll ends).

Timeframe

Opening time: The opening time of the poll. No one will be able to cast a vote before this time.

Closing time: The closing time of the poll. No one will be able to cast a vote after this time. The final results are accessible only after this time has passed.

Periodic poll: If set, the poll will be re-run regularly without the need to explicitly recreate it. Unless participants change their votes, their previous votes will be considered as still valid. Therefore, this feature mostly makes sense for private polls where the voters can change their votes.

Period: Periodicity of the poll. If the period of the poll is set to x days, the poll will be repeated every x days.

Input and output

Pnyx supports a range of different preferences to be elicited from the voters as well as different output formats (see the full descriptions for more details). Based on these choices a suitable aggregation method will be chosen by Pnyx (see the aggregation rules).

Alternatives

Alternative: Each poll has to specify a set of feasible alternatives. Each alternative is specified by its name and an optional description.

Tie breaking rule: If required, tie breaking is performed by pnyx according to a specified order over the alternatives. It can be defined randomly or manually by the poll creator. This tie breaking is used for every choice rule except Nash's rule, Maximal lotteries and Kemeny's rule. For Nash's rule and Kemeny's rule the tie breaking is performed during the computation by the LP solver. For maximal lotteries, the barycenter of the space of all maximal lotteries is approximated.

Participants

For private polls, each participant is invited by an email from the Pnyx system. For public polls, this is optional (alternatively the poll creator can share the public link to the poll). Pnyx furthermore provides a built-in interface for the poll admin to send emails to all participants at any time.

The email addresses entered will not be shared with third parties or otherwise be used for commercial purposes.

Input types (individual preferences)

Most preferred alternative: The voter can only select a unique alternative among all candidates. This is how in practice elections are often carried out.

Approved alternatives: The voter may approve a set of alternatives among the candidates. The technical term are dichotomous preferences. There is no distinction between alternatives within the set of approved or non-approved alternatives, respectively.

Rankings without ties: Each voter has to rank all alternatives and the ballot may not contain ties between alternatives. Such preferences are also called linear.

Rankings with ties: This input format is a generalization of "rankings without ties" where now ties between alternatives are allowed. One speaks of complete preorders or bucket preferences.

Pairwise comparisons: This is the most general form of preferences that are supported by Pnyx. The voter may specify each individual pairwise comparison among alternatives. By default, indifference between any pair is assumed. Note that transitivity of the preferences is no longer required.

Output types (collective preferences)

Unique winner: The result of the poll is a unique best alternative.

Lottery: The result of the poll is represented by a probability distribution over the alternatives. This output format allows for particularly nice aggregation rules with good theoretical properties. Note that it is also a suitable format when a divisible resource should be allocated/shared (e.g., a meeting room), where then the probabilities stand for shares of the resource.

Rankings without ties: The collective preferences are provided by a complete ranking of all alternative without ties. This can also be interpreted as a collective order of preference over the alternatives.

Aggregation rules:

After successful collection of the preferences of all voters, those preferences are aggregated into a single collective preference of the group. For this an aggregation rule depending on the desired input & output types is applied.

The Pnyx project aims to keep things simple for the user, why the aggregation rules to be used have been preselected based on scientific insights.

The following table gives an overview of the implemented aggregation rules and for which combination of input and output which rule is applied.

Plurality rule: In this voting system the winner is the alternative with the most votes (recall that each voter may only vote for a single alternative); there is no requirement that the winner gains an absolute majority of votes.

Approval voting: Each voter may "approve" (i.e., select) as many alternatives as desired. The winner is the alternatives with most approval votes.

Borda's rule: Borda's rule is a scoring rule where each alternative receives from 0 to m - 1 points from each voter (depending on the position the alternative is ranked in), where m is the total number of alternatives. The alternative with the highest cumulative score is selected.

Bucket Borda's rule: Bucket Borda's rule is an extension of Borda's rule allowing for ties in voters' preferences. Among a set of tied alternatives each alternative receives the average score of all possible orderings of those alternatives. In total each voter still gets to assign the same number of points (regardless of how many ties he specifies).

Young's generalization of Borda's rule: This rule generalizes both Borda's as well as Bucket Borda's rule and only requires pairwise comparisons of alternatives. For each voter, each alternative receives 1 point for every alternative it dominates and loses 1 point for each alternative which it is dominated by.

Random dictatorship: This lottery corresponds to selecting a single voter at random and letting him choose among the alternatives. It has the nice property that "each vote counts".

Nash's rule: Nash's rule maximizes a convex collective utility function and guarantees to every non-indifferent voter a lower bound of satisfaction. For more details please refer to A. Bogomolnaia, H. Moulin, and R. Stong. Collective choice under dichotomous preferences. Journal of Economic Theory, 122(2):165–184, 2005.

Maximal lotteries: Maximal lotteries are an advanced aggregation method equivalent to mixed maximin strategies of the symmetric zero-sum game given by the pairwise majority margins. Whenever there is an alternative that is preferred to any other alternative by some majority of voters (a so-called Condorcet winner), the lottery that assigns probability one to this alternative is a maximal lottery. This is in contrast to random dictatorship, which only returns a lottery with support one if all voters unanimously favor the same alternative. At the same time, maximal lotteries satisfy consistency with respect to variable electorates which has been identified as the defining property of Borda’s scoring rule. For more detail please refer to H. Aziz, F. Brandt, and M. Brill. On the tradeoff between economic efficiency and strategyproofness in randomized social choice. In Proceedings of the 12th International Conference on Autonomous Agents and Multi-Agent Systems (AAMAS), pages 455–462. IFAAMAS, 2013.

Plurality scores: This rule yields a linear ranking based on plurality scores (see above).

Approval voting scores: This rule yields a linear ranking based on approval scores (see above).

Kemeny's rule: This rule yields a linear ranking that agrees with as many voters' pairwise preferences as possible.
